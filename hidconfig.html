⚠ Connected over USB, but the Pi isn't answering config requests.
The hidconfig service (s0_hidconfig.py) is not running — and if it's down, the engine likely is too. On the Pi run:
systemctl cat expander hidconfig | grep ExecStart — both must point at the new s6_orchestrator.py / s0_hidconfig.py. If not:
sudo cp ~/expander/*.service /etc/systemd/system/ && sudo systemctl daemon-reload && sudo systemctl restart expander hidconfig
Then check journalctl -u expander -n 20 for Python errors and reconnect here.

Overview
Assignments (add / edit / remove)

Each row is one input. Pick its Input and Output; the layer columns show what fires on each layer (click to change). Group is optional. ✕ removes the row.

Devices

Every input device is an entry here -- there are no built-ins. A device's Baseline maps its physical keys to your standard layout; all Custom Mappings (layers, chords, combos, macros) are shared and apply identically on every device. Pin = any unique part of the device name shown by the Bluetooth tab or RAW log; one entry covers all of a device's interfaces (keyboard + trackpad).

Then ⬆ Save to device; the engine picks up new devices on service restart.

Expansion sets

One file per set on the Pi (expansion_sets/). Toggle a set's checkbox to (de)activate it; the engine hot-reloads within ~2 s of ⬆ Save sets. Regex items support $1–$9 captures, {{#…}} inline calculation, {YYYY}-{MM}-{DD} date math and {^} cursor placement.

Date & time triggers

These are built into the engine (not stored in expansions.json) -- edit the trigger text below, then use the normal ⬆ Save to device button at the top to apply.

Expansions now also auto-load when you Connect.

Triggers like \sig; values may embed chords: {{ctrl+c}}. Date/time trigger text is set above, not hardcoded to \dt \tm \tdt anymore.

Expressions (implemented in Python on the Pi — documented here)

Scroll acceleration (layer 0 wheel): momentum v = v·0.98^dt_ms + detents, output v + 0.82·(1.4v)³, clamped ±150, slow-scroll halved below 3.1 and again below 2.0, fractional output accumulated. Constants: ACCEL_DECAY, ACCEL_CUBIC_GAIN, ACCEL_CUBIC_SCALE, ACCEL_CLAMP, ACCEL_SLOW1, ACCEL_SLOW2 at the top of expander.py.

Turbo click (CapsLock + hold left): 20 Hz, 50 ms period (TURBO_PERIOD).

DPI cycle (software pointer scale): factors [1.0, 1.5, 2.0, 0.5] (DPI_FACTORS), applied with fractional accumulation.

Active OS: win

One profile, many OSes. The base columns are what runs by default; each OS column holds only the differences for that OS (a blank cell = same as base). Click a cell to set an OS-specific override; the ⇄ reorders which OS is base.

Per-device scrolling

Tick to invert scroll direction for a device on the selected OS (e.g. natural on Mac, reversed on PC).

Unicode typing method (how ☑ / Δ are typed)

macOS needs the “Unicode Hex Input” source enabled & selected for mac_option_hex. Windows win_alt_hex needs the EnableHexNumpad registry setting + Num Lock.

Macro / mapping differences

Rows are macros & layer mappings that differ (or can differ) per OS. Base is the default; set an OS cell to override just for that OS. Empty = inherits base.
